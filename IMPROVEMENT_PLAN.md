## 📊 แผนการปรับปรุง 4 กลยุทธ์

### 🥇 Priority 1: เปลี่ยนโมเดล Embedding (BGE-M3)

**ระดับความสำคัญ:** ⭐⭐⭐⭐⭐ สูงสุด  
**เวลาที่ต้องใช้:** 10-15 นาที  
**ผลลัพธ์คาดหวัง:** +5-8% accuracy

```
ปัญหา:
  - mxbai-embed-large ออกแบบสำหรับภาษาอังกฤษ
  - ไม่เข้าใจภาษาไทยอย่างดี
  - Semantic matching อ่อนกำลัง

แนวทางแก้:
  - ใช้ BGE-M3 (Baidu) - เป็น multilingual model
  - ได้รับการฝึกอบรมกับข้อมูล Thai ในปริมาณใหญ่
  - ติดตั้งแล้วใน Ollama

ขั้นตอน:
  1. แก้ไข config.py: EMBEDDING_MODEL = "bge-m3"
  2. ทดสอบ 10 คำถามแรก
  3. ถ้าสดใจ → ทดสอบแบบเต็ม 100 คำถาม
  4. บันทึกผลลัพธ์

ผลลัพธ์ที่คาดหวัง:
  Old: 0.6611 (Supabase)
  New: ~0.7108 (+5-8%)
```

---

### 🥈 Priority 2: เพิ่มจำนวนเอกสารที่ค้นหา (top_k)

**ระดับความสำคัญ:** ⭐⭐⭐⭐ สูง  
**เวลาที่ต้องใช้:** 5 นาที  
**ผลลัพธ์คาดหวัง:** +3-5% accuracy

```
ปัญหา:
  - top_k=3 ดึงเอกสารเพียง 3 อย่าง
  - บางครั้งเอกสารที่เกี่ยวข้องตกหล่น
  - บริบทไม่เพียงพอสำหรับคำตอบที่ดี

แนวทางแก้:
  - เพิ่ม top_k จาก 3 → 7
  - ดึงเอกสารมากขึ้นให้โมเดล LLM เลือก
  - โมเดล LLM จะสร้างคำตอบที่ดีขึ้น

ขั้นตอน:
  1. แก้ไข test_dual_mode.py: top_k: 3 → 7
  2. ทดสอบ 10 คำถาม
  3. ถ้าสดใจ → ทดสอบแบบเต็ม 100 คำถาม
  4. เปรียบเทียบผลลัพธ์

ผลลัพธ์ที่คาดหวัง:
  Old: 0.6611
  New: ~0.6911 (+3%)
  ⚠️  ช้าลง 50% (เนื่องจากดึงเอกสารมากขึ้น)
```

---

### 🥉 Priority 3: เพิ่มขั้นตอน LLM Re-ranking

**ระดับความสำคัญ:** ⭐⭐⭐⭐⭐ สูงสุด (ผลลัพธ์ดีที่สุด)  
**เวลาที่ต้องใช้:** 30-45 นาที  
**ผลลัพธ์คาดหวัง:** +8-12% accuracy ⭐⭐⭐

```
ปัญหา:
  - เอกสารที่คล้ายกันเวกเตอร์ อาจไม่เกี่ยวข้องจริง ๆ
  - โมเดล LLM ใช้เอกสารแรกบน ไม่ถูกคัด
  - ต้องการหลังจากค้นหา re-rank ใหม่

แนวทางแก้:
  Step 1: ค้นหา top_k=10 เอกสารผ่าน vector
  Step 2: ใช้ LLM ให้ score ความเกี่ยวข้อง 0-100
  Step 3: เลือก top_3 re-ranked เพื่อสร้างคำตอบ
  
  ตัวอย่าง:
    Question: "ภาษามือมีกี่แบบ"
    
    Vector ค้นหา: [Doc1 (0.75), Doc2 (0.70), Doc3 (0.68)]
    
    LLM re-rank:
      - Doc1: relevance 95/100 ✓
      - Doc2: relevance 45/100 (ไม่เกี่ยวข้องจริง)
      - Doc3: relevance 88/100 ✓
    
    ผลลัพธ์: [Doc1 (95), Doc3 (88)] → สร้างคำตอบ

ขั้นตอน:
  1. สร้าง function rerank_documents()
  2. ผสมเข้า answer_question()
  3. ทดสอบ 10 คำถาม
  4. ถ้าสดใจ → ทดสอบแบบเต็ม

ผลลัพธ์ที่คาดหวัง:
  Old: 0.6611
  New: ~0.7422 (+8-12%) ⭐ ดีที่สุด
  ⚠️  ช้ามากขึ้น (-6-9 วินาทีต่อคำถาม) แต่ผลลัพธ์ดี
```

### 🏅 Priority 4: ปรับปรุงการแบ่งเอกสาร (Thai Chunking)

**ระดับความสำคัญ:** ⭐⭐⭐ ปานกลาง  
**เวลาที่ต้องใช้:** 1 ชั่วโมง+  
**ผลลัพธ์คาดหวัง:** +2-4% accuracy

```
ปัญหา:
  - แบ่งหลาย ๆ paragraph แต่ไม่สนใจภาษาไทย
  - ไม่มี tokenizer สำหรับภาษาไทย
  - ความหมายอาจตัดขาดตรงจุดแบ่ง

แนวทางแก้:
  - ใช้ pythainlp (Thai tokenizer)
  - แบ่งตามประโยค Thai ไม่ใช่ paragraph
  - ความหมายสมบูรณ์มากขึ้น

ขั้นตอน:
  1. pip install pythainlp
  2. ปรับปรุง vault_embeddings_cache.json
  3. ปรับปรุง local_vector_store.py
  4. ทดสอบใหม่

ผลลัพธ์ที่คาดหวัง:
  Old: 0.6611
  New: ~0.6811 (+2-4%)
  ℹ️  ต้องทำใหม่ทั้งหมด (เวลามากที่สุด)
```

## 📅 แผนการดำเนินการ

