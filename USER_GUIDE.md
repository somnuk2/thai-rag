# 📘 คู่มือการใช้งานระบบ Local RAG สำหรับภาษามือไทย (User Guide)

ระบบนี้คือ **Local RAG (Retrieval-Augmented Generation)** ที่ออกแบบมาเพื่อให้อ่านและตอบคำถามจากเอกสาร PDF ภาษาไทย (เน้นด้านภาษามือไทย) โดยประมวลผลทุกอย่างในเครื่องคอมพิวเตอร์ของคุณเองเพื่อความเป็นส่วนตัว

---

## 🏗 โครงสร้างโปรเจกต์ (Project Structure)

*   **`localrag.py`**: โปรแกรมหลักสำหรับพูดคุยกับ AI (Chat) และดึงข้อมูลจากเอกสาร
*   **`process_thai_pdf.py`**: สคริปต์สำหรับอ่านไฟล์ PDF ภาษาไทยและนำข้อมูลเข้าสู่ระบบ
*   **`vault.txt`**: ไฟล์เก็บข้อมูลดิบที่สกัดออกมาจากหนังสือ
*   **`run_eval.py`**: สคริปต์สำหรับทดสอบความแม่นยำของระบบอัตโนมัติ
*   **`original_code/`**: โฟลเดอร์เก็บไฟล์ต้นฉบับที่ไม่เกี่ยวข้องกับการทำงานหลักในปัจจุบัน

---

## 🛠 สิ่งที่ต้องเตรียม (Prerequisites)

1.  **Python 3.10+**
2.  **Ollama**: ต้องติดตั้งและเปิดใช้งานอยู่เสมอ (ดาวน์โหลดที่ [ollama.com](https://ollama.com))
3.  **Models ใน Ollama**:
    *   `gemma3:4b` (หรือ `llama3`) สำหรับตอบคำถาม
    *   `mxbai-embed-large` สำหรับการค้นหาบทความ (Embeddings)

---

## 🚀 ขั้นตอนการใช้งาน

### 1. การนำเอกสารเข้าสู่ระบบ (Ingest Data)
หากต้องการเพิ่มหนังสือ PDF เล่มใหม่เข้าไปในระบบ:
1.  นำไฟล์ PDF ไปวางในโฟลเดอร์โปรเจกต์
2.  รันคำสั่งสกัดข้อความ:
    ```powershell
    python process_thai_pdf.py
    ```
    *ข้อความจะถูกเพิ่มเข้าไปใน `vault.txt` โดยอัตโนมัติ*

### 2. การเริ่มใช้งานแชท (Chatting)
คุณสามารถเริ่มถาม-ตอบกับ AI เกี่ยวกับเนื้อหาในหนังสือได้ด้วยคำสั่ง:
```powershell
python localrag.py --model gemma3:4b
```
**ฟีเจอร์เด่น:**
*   **Embedding Cache**: ในการรันครั้งแรกระบบจะคำนวณรหัสข้อมูลไว้ ครั้งต่อไปจะเปิดโปรแกรมได้เร็วขึ้นทันที
*   **--clear-cache**: หากคุณแก้ไข `vault.txt` ด้วยมือ และต้องการให้ AI อัปเดตข้อมูลใหม่ ให้รัน:
    ```powershell
    python localrag.py --model gemma3:4b --clear-cache
    ```

### 3. การประเมินความถูกต้อง (Evaluation)
หากต้องการวัดผลว่าระบบตอบคำถามแม่นยำแค่ไหน:
1.  **สร้างชุดคำถามอ้างอิง (Ground Truth)** (ถ้ายังไม่มี):
    ```powershell
    python generate_ground_truth.py
    ```
2.  **รันการทดสอบ 100 ข้อ**:
    ```powershell
    python run_eval.py
    ```
3.  **ดูผลลัพธ์**: เปิดไฟล์ `evaluation_results.json` เพื่อดูคะแนนเฉลี่ยและความถูกต้องในแต่ละข้อ

---

## 📈 เกณฑ์การให้คะแนน (Evaluation Metrics)

ระบบใช้ AI ในการตรวจข้อสอบ (LLM Judge) โดยให้คะแนน 1-5 ดาว:
*   ⭐ **5/5**: ถูกต้องแม่นยำตามหนังสือ
*   ⭐ **3/5**: ตอบถูกบางส่วน หรือดึงเนื่อหามาปนกันเยอะ
*   ⭐ **1/5**: ตอบผิด หรือหาข้อมูลไม่เจอ

---

## 📝 ข้อแนะนำเพิ่มเติม
*   **ความเร็วในการตอบ**: ขึ้นอยู่กับสเปกเครื่อง (GPU) และขนาดของ Model ที่เลือกใช้
*   **การเพิ่มความแม่นยำ**: หาก AI ตอบไม่ตรง ควรตรวจสอบความสะอาดของข้อมูลใน `vault.txt` ว่ามีตัวอักษรขยะปนอยู่เยอะหรือไม่
